Modernizing a Federal Business Operating System

Summary

As the sole designer on a 50+ person program, I led UX research, design systems, and UI modernization efforts across multiple mission-critical internal applications.

I established the program’s first UX research initiative, created a scalable cross-application design system, and partnered with engineering to modernize legacy interfaces and frontend architecture.

Problem

A federal defense client relies on our team to maintain and evolve internal applications supporting account management, budgeting, and operational workflows.


However, these applications were:

  • Built by fragmented teams over time
  • Inconsistent in UX patterns and accessibility
  • Running on outdated frontend frameworks
  • Accumulating significant UX and technical debt

In short, there was no centralized UX governance or shared design system.

Goals

  1. Establish a scalable, cross-application design system
  2. Modernize legacy UIs alongside required technical migrations
  3. Identify and resolve usability friction as the sole user advocate on the team
01

Part 1: Initial User Research

Before redesigning, I assessed how the Access Management Application was actually used and where friction occurred.

Research Approach

Since direct access to end users was limited, I took the following approaches to conducting user research:

  • End-to-end site audit
  • Review of support tickets
  • Collaboration with helpdesk staff
  • Affinity mapping recurring issues

Affinity Map

Key Findings

  • Permission updates were destructive by default, forcing users to rebuild configurations
  • Navigation and IA were inconsistent and cognitively heavy
  • Visual patterns and color contrast failed modern accessibility standards
  • Users developed workarounds outside the system

Overall, the product suffered from accumulated UX debt. Systemic issues, not isolated screens, required a program-level solution.

Part 3: Application Modernization

Enterprise Portal UI Refresh

The portal was intended to be a central resource hub, and the entry point to all applications. However it was underutilized. Users bypassed it via bookmarks and described it as unintuitive.


My proposal was approved and reinforced the need for UX governance, though implementation was later deprioritized due to shifting priorities.

Access Management App UI Refresh

The frontend framework had reached end-of-life, requiring migration. I positioned this as an opportunity to modernize both technology and UX simultaneously.


My Process:

  1. Researched and compared modern frontend frameworks
  2. Evaluated maintainability, ecosystem support, security, and migration effort
  3. Partnered with engineering to align on feasibility
  4. Assessed integration with the emerging design system

Outcome

The team aligned on migrating to React with a component-based architecture and UI library to accelerate development and standardize patterns. This migration:

  • Improved maintainability and scalability
  • Standardized UI components
  • Increased accessibility compliance
  • Reduced frontend technical debt
04

Conclusion

This work established foundational UX infrastructure across the program.

Isolated Audits Evolved Into...

  • A shared design system adopted across applications
  • Clear usability and accessibility standards
  • Stronger design–engineering alignment
  • A scalable foundation for frontend modernization

As the sole designer, I didn’t just redesign interfaces — I introduced systems, governance, and UX maturity into an environment that previously had none.
